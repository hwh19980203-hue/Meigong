# 设计交付规范

> 本文档定义设计产出的最终交付标准，独立于具体的工作流流程，适用于所有设计需求类型。

---

## 交付前终质检清单

每套设计交付前，逐项确认：

### 图片质量
- [ ] 所有图片**清晰无锯齿**
- [ ] 产品颜色与实物**无明显偏差**
- [ ] 品牌色值**准确**
- [ ] 图片中的英文文字**拼写正确**
- [ ] 品牌 Logo **完整、未变形**

### 合规复查
- [ ] 构图无**实质性相似**风险
- [ ] 字体具有**商业授权**
- [ ] 无他人**商标词**植入
- [ ] 产品外观无**专利混淆**风险
- [ ] 已完成合规审查记录存档

### 技术规范
- [ ] 分辨率 ≥ **1000×1000 px**（建议 2000×2000）
- [ ] 格式正确（JPEG / PNG / PSD）
- [ ] 首图背景纯白 **RGB 255,255,255**
- [ ] 图片尺寸比例正确（1:1 / 其他）

---

## 交付格式标准

| 用途 | 格式 | 说明 |
|------|------|------|
| **亚马逊上传** | JPEG | 首选格式，文件小、兼容性好 |
| **需要透明背景** | PNG | 首图、抠图素材 |
| **可编辑源文件** | PSD | 分层文件，保留商品主体/背景/文字/Logo/模特各层 |
| **模板存档** | PSD / AI | 保留可复用的设计模板 |

## 文件命名规范

### 通用格式

```
[ASIN]-[模块]-[版本号].[格式]
```

| 字段 | 说明 | 示例 |
|------|------|------|
| ASIN | 产品标识 | B0ABCDEFG |
| 模块 | 图片类型 | main / scene / aplus / infographic / ad |
| 版本号 | v + 数字 | v1 / v2 / v3-final |

### 模块缩写对照

| 缩写 | 含义 |
|------|------|
| `main` | 白底主图 |
| `scene` | 场景图 |
| `infographic` | 卖点/功能图 |
| `detail` | 细节图 |
| `size` | 尺寸图 |
| `step` | 步骤图 |
| `compare` | 对比图 |
| `aplus-01` ~ `08` | A+ 模块 01-08 |
| `ad-sb` | Sponsored Brands 广告 |
| `ad-sd` | Sponsored Display 广告 |
| `pack` | 包装图 |
| `brand` | 品牌图 |

### 完整示例

```
B0ABCDEFG-main-v1.jpg          → 首版白底主图
B0ABCDEFG-main-v3-final.jpg    → 终版白底主图
B0ABCDEFG-aplus-01-v2.jpg      → A+ 模块 01 第二版
B0ABCDEFG-ad-sb-v1.jpg         → Sponsored Brands 广告初版
```

## 归档结构

交付文件按以下目录结构归档：

```
交付归档/
└── [ASIN]/
    ├── images/          ← 上传用的最终图片
    │   ├── main.jpg
    │   ├── scene-01.jpg
    │   └── aplus-01.jpg
    ├── psd/             ← 可编辑源文件
    │   ├── main.psd
    │   └── aplus.psd
    └── reference/       ← 参考素材
        ├── product-photos/
        └── competitor-ads/
```

## 交付流程

```
终质检通过
    ↓
格式转换（JPEG / PNG / PSD）
    ↓
按命名规范重命名文件
    ↓
按归档结构整理到交付目录
    ↓
上传至运营需求平台
    ↓
通知组长确认交付完成
```

## 常见问题

### Q：应该用 JPEG 还是 PNG？
→ 亚马逊上传首选 **JPEG**（文件小、加载快）。需要透明背景的场景用 PNG。源文件一律保留 PSD。

### Q：版本号怎么管理？
→ 初版用 `v1`，每次修改递增。最终确认版在末尾加 `-final`。废弃版本不要删除，保留在历史记录中。

### Q：给运营交付时需要注意什么？
→ 确保文件名规范可识别、图片格式正确、压缩质量不损失。最好附带一份简短的交付说明（包含文件清单和修改记录）。
