# AI 驱动美工工作流

> 本文档定义美工部门的 **AI + 人工协作模式**：AI 负责批量生成主图/场景图/A+ 配图等 80% 的工作量，人工负责细节审核、精修调色和最终交付。

---

## 核心理念

```
AI 负责：批量产出、快速迭代、多方案探索
人工负责：合规把关、审核精修、品牌一致性、创意决策
```

## 详细工作流

### 第一阶段：需求接入

| 步骤 | 负责人 | 内容 |
|------|--------|------|
| 1.1 接收需求 | 美工组长 | 确认需求类型（新 Listing / A+ 更新 / 广告素材 / Brand Store） |
| 1.2 收集素材 | 运营/产品 | 提供：产品实物、多角度照片、卖点文档、品牌规范、参考图 |
| 1.3 需求登记 | 美工组长 | 在任务管理工具中登记，附上所有素材 |
| 1.4 标准输入表 | 需求方 | 填写 [[wiki/templates/AI作图任务标准输入表]]，确保 AI 能准确理解任务 |

> **标准输入表**：[[wiki/templates/AI作图任务标准输入表]] — 用于标准化作图需求输入，包含产品信息、视觉要求、图片清单、合规标注等字段。所有 AI 作图任务应优先通过此表输入。

### 第二阶段：AI 批量产出（由 LLM/AI 工具执行）

此阶段覆盖 **80% 的设计工作**，AI 工具（ChatGPT Image 2 / Midjourney / 其他）负责：

#### 2.1 产品特征分析

将产品多角度照片 + 以下信息发给 AI：

```
产品名称：[名称]
ASIN：[ASIN]
主要材质：[材质]
核心尺寸：[尺寸]
核心卖点：[卖点1、卖点2、卖点3]
目标站点：[美国/欧洲/日本]
品牌色：[色值]
竞品参考：[参考图]
```

AI 输出结构化的产品特征描述，人工确认后进入下一步。

#### 2.2 主图生成

**白底主图**（1 张）：
```
提示词框架：产品精修 + 白底 + 正视图 + [材质描述] + 清除瑕疵 + 电商主图标准
```

**场景主图**（2-3 张方案）：
```
提示词框架：场景描述 + 模特要求 + 产品摆放 + 光影风格 + 品牌调性
```

#### 2.3 A+ 配图生成（6-8 张）

| 模块 | 内容 | AI 生成要点 |
|------|------|-------------|
| 01 英雄图 | 产品全景 + 场景 | 高品质场景，突出产品整体 |
| 02 卖点图 | 核心功能标注 | 排版布局 + 英文标注 + 图标 |
| 03 细节图 | 材质/工艺特写 | 微距视角 + 质感强调 |
| 04 对比图 | 使用前后 / vs 竞品 | 分屏布局 + 视觉对比 |
| 05 尺寸图 | 产品尺寸标注 | 尺寸线标注 + 简洁排版 |
| 06 场景图 | 使用场景 | 生活方式场景 + 人物互动 |
| 07 包装图 | 包装内容展示 | 产品全家福 |
| 08 品牌图 | 品牌故事/承诺 | 品牌色 + Logo + Slogan |

#### 2.4 广告素材生成（2-3 张）

- Sponsored Brands 创意素材
- Sponsored Display 素材
- 站外广告素材

#### 2.5 AI 自动检查清单

AI 在交付前自查以下项目，**标记通过/不通过**：

- [ ] **产品一致性**：产品外形、颜色、细节是否与实物一致
- [ ] **比例正确**：产品在画面中的比例是否合理（基于给出的尺寸）
- [ ] **文字拼写**：所有英文文字拼写正确
- [ ] **Logo 完整**：品牌 logo 未被 AI 篡改或变形
- [ ] **无 AI 幻觉**：没有多余的产品元素（如多的螺丝、配件）
- [ ] **颜色准确**：产品颜色与实物无明显偏差
- [ ] **分辨率**：满足亚马逊图片要求（≥ 1000×1000，建议 2000×2000）

### 第三阶段：合规审查

> **核心原则**：AI 生成的内容并非"天然安全"。即使 AI 重新渲染的图片，仍可能因套用竞品构图、未经授权字体等触发亚马逊侵权下架。
> 详细合规指南见 [[wiki/concepts/AI内容侵权风险与合规]]。

#### 3.1 合规检查清单

| 检查项 | 检查内容 | 方法 |
|--------|----------|------|
| **构图相似性** | 场景图/A+ 是否与竞品实质性相似（背景、布局、视角） | 人工比对参考图 |
| **字体授权** | 图片中使用的字体是否有商业授权 | 确认字体来源 |
| **图案元素** | 图片中的装饰图案/图标是否原创 | 人工判断 |
| **外观专利** | 产品整体观感是否与他人设计专利混淆 | USPTO/EUIPO 检索 |
| **商标词** | 图片文字/A+ 文案中是否包含他人注册商标 | 侵权词工具过滤 |
| **证据存档** | 保存 AI 提示词 + 产出截图 + 产品原图 | 归档备查 |

#### 3.2 合规问题处理

- 发现侵权风险 → **立即停止使用该素材**，修改或重新生成
- 修改后仍存疑 → 咨询法务或知识产权顾问
- 所有合规结果 → 记录到任务存档中备查

### 第四阶段：人工精修

| 步骤 | 负责人 | 内容 | 耗时参考 |
|------|--------|------|----------|
| 4.1 初审核查 | 美工组长 | 审查 AI 产出的所有图片，标注需要修改的图 | 15-30 min/套 |
| 4.2 颜色校准 | 美工 | PS 调色，确保产品颜色与实物一致 | 10-20 min/套 |
| 4.3 细节修正 | 美工 | PS 修复 AI 幻觉（多/少的螺丝、变形等） | 10-15 min/套 |
| 4.4 文字排版 | 美工 | 人工确认/调整所有文字标注 | 10-15 min/套 |
| 4.5 批量 PS 导出 | 美工 | 整理图层、导出最终交付文件 | 5-10 min/套 |

### 第五阶段：交付

按 [[wiki/standards/设计交付规范]] 执行终质检、格式转换、命名归档、上传交付。

## 第六阶段：知识回写

> 交付不是终点。将产出和经验反哺知识库，形成"使用 → 沉淀 → 复用"的闭环。

### 6.1 回写案例库

每次交付后，将本次设计案例添加到 [[wiki/cases/index|案例库]]：

| 记录项 | 内容 |
|--------|------|
| 需求背景 | 本次设计的需求描述 |
| 出图方案 | 使用的标准输入表 + prompt |
| 效果图 | 最终交付图片（嵌入或链接） |
| 亮点/踩坑 | 什么做得好、什么需要改进 |
| 经验标签 | 如"多色产品排版"、"场景图光影问题" |

### 6.2 更新产品知识

更新 [[wiki/products/index|产品知识库]] 中对应产品页面：

- 补充本次使用的 **视觉策略** 和 **风格参考**
- 记录有效的 **prompt 模式** 和 **参数组合**
- 标注本次遇到的 **问题和解决方案**

### 6.3 反哺模板与规范

- 如果发现了新的有效的排版/布局，考虑更新对应 [[wiki/templates/index|模板]]
- 如果发现了规范盲区（如亚马逊新规），更新对应规范页面，如 [[wiki/standards/亚马逊主图规范]] 或 [[wiki/standards/A+页面设计规范]]
- 如果沉淀了新的提示词模式，更新本工作流的提示词模板库

### 6.4 回写检查清单

- [ ] 案例已归档到 `wiki/cases/`
- [ ] 产品页的视觉策略已更新
- [ ] 模板/规范是否需要更新（如需要，已更新或已创建待办）
- [ ] 合规经验已记录

## AI vs 人工：职责划分对照表

| 设计环节 | AI 承担 | 人工承担 | AI 节省时间 |
|----------|---------|---------|------------|
| 产品特征分析 | 初稿撰写 | 审核修正 | ~30 min |
| 白底主图 | 完整生成 | 颜色微调 | ~60 min |
| 场景主图 | 多方案生成 | 选稿 + 精修 | ~90 min |
| A+ 配图（8张） | 批量生成 | 逐张审查微调 | ~4 小时 |
| 广告素材 | 初稿生成 | 品牌调性把关 | ~60 min |
| 图片精修 | 提示词精修 | 最后质感调校 | ~30 min |
| 竞品反推 | 版式分析 + 复刻 | 创意决策 | ~30 min |
| **合规审查** | AI 辅助筛查 | **人工最终判定** | ~15 min |
| **合计** | **~80% 工作量** | **~20% 工作量** | **以前 2-3 天 → 现在 2-4 小时** |

## 不同需求类型的差异流程

### 类型 A：全新 Listing 主图 + A+

```
AI 产出 → 合规审查 → 组长初审 → 美工精修 → 按交付规范执行 → 回写知识库
预估总耗时：2.5-4.5 小时（传统方式 2-3 天）
```

### 类型 B：A+ 更新（产品图已有）

```
AI 直接基于现有产品图生成 A+ → 合规审查 → 美工简单调色 → 按交付规范执行 → 回写知识库
预估总耗时：1.5-2.5 小时（传统方式 1-2 天）
```

### 类型 C：紧急广告素材

```
AI 生成 3-5 个方案 → 合规审查 → 组长选稿 → AI 调整 → 按交付规范执行 → 回写知识库
预估总耗时：45-75 分钟（传统方式半天）
```

### 类型 D：竞品视觉分析

```
投放竞品图 → AI 反推版式 Prompt → 组长评估可行性 → AI 生成测试 → 决定方向 → 记录到案例库
预估总耗时：1-2 小时（传统方式需多次尝试）
```

## 提示词模板库

### 白底主图模板
```
产品精修，[产品名称]置于纯净的纯白背景上。[视角描述]，[3D渲染/实拍质感]，
精准还原[材质描述]。清除所有指纹、灰尘与瑕疵，让产品看起来崭新。
[材质]光影立体感强，提升整体质感和高级感。
产品上的文字/标签需清晰锐利。光线柔和均匀，符合电商主图标准。
```

### 场景主图模板
```
实拍风格场景图，[场景描述：如户外/室内/厨房]。[模特描述：年龄/人种/动作/服饰]，
正在使用[产品]。[背景描述]。[光线描述：自然光/暖光/柔光]。
整体氛围[氛围描述：温馨/高端/专业]，真实感强，不过度完美。
图片尺寸 1:1。
```

### A+ 卖点图模板
```
[风格参考图]，[排版参考图]。
主要要求：[场景描述]。
[模特描述]。
文字标注：[大标题] / [副标题] / [小字说明]。
字体风格：[字体风格]，英文字体，清晰锐利。
图标要求：[图标描述]。
图片尺寸 1:1，文字区域预留充足。
```

### PSD 导出模板
```
帮我把这张电商商品图精准分层分离，分别拆分出：
商品主体、背景、文字、品牌logo、模特，
每层独立分开，清晰干净，无杂边。PSD格式。
```

## 团队角色与技能要求

### AI 操作员（可由现有美工转型）
- 熟练掌握 ChatGPT Image 2 提示词编写
- 理解产品材质、光影、构图等视觉语言
- 能够将设计师的需求转化为 AI Prompt

### 精修美工（保留核心技能）
- Photoshop 调色与修图能力
- AI 幻觉修复（多/少的细节修正）
- 品牌一致性把控

### 审核组长（流程管理）
- AI 产出质量把控
- 修改反馈的标准化传达
- 流程效率优化

## 常见问题处理

### Q：AI 生成的图片产品颜色不对怎么办？
→ 交给美工 PS 调色，同时在提示词中标注品牌色值（RGB/HEX），逐步优化 AI 的颜色还原能力。

### Q：AI 生成的文字拼写错误？
→ 在提示词中强调文字内容并说明"请确保拼写完全正确"，但**文字最终仍需要人工核对**。

### Q：AI 在产品上添加了不存在的螺丝/配件？
→ 这是 AI 幻觉，需要在提示词中明确列举产品包含和不包含的部件，后续由美工 PS 修正。

### Q：如何保持多张场景图的产品一致性？
→ 使用 AI 的全景图功能，或提供同一组产品多角度照片作为垫图，确保产品外观一致。

### Q：新人如何快速上手这套流程？
→ 先学习提示词模板库，从白底主图开始，逐步过渡到场景图、A+。每类需求都有标准模板可套用。

### Q：AI 生成的图片会不会侵权？
→ 有可能！详见 [[wiki/concepts/AI内容侵权风险与合规]]。核心原则：AI 生成的每一张图都需要经过合规审查——检查构图是否与竞品相似、字体是否有授权、产品外观是否涉及他人专利。

### Q：合规审查具体要查什么？
→ 查 6 项：构图相似性、字体授权、图案元素原创性、外观专利、商标词、证据存档。建议固定由组长或专人负责这一环节。

## 迭代优化

本工作流应每周回顾优化，重点关注：

1. **AI 产出通过率**：每次人工审查标注的修改项，归类统计，优化提示词减少同类问题
2. **单套耗时追踪**：记录每套图的 AI + 人工耗时，持续压缩
3. **提示词迭代**：将频繁出现的修图项反推为提示词约束条件
4. **新工具接入**：关注 AI 生图工具的更新（如 GPT-4o 的新功能），及时纳入流程
5. **合规零事故**：确保每套图都经过合规审查，记录审查结果
6. **回写完成率**：统计每次交付后案例库和产品页的更新情况，确保知识闭环
