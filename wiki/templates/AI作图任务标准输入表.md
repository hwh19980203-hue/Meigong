# AI 作图任务标准输入表

> 本表用于把运营、产品、开发、美工的作图需求整理成 Codex 和 GPT 能稳定执行的标准输入。每个 AI 作图任务开始前，优先填写本表。

## 使用方式

1. 需求方复制下面的“任务输入表”
2. 尽量填写必填项
3. 把产品图片、模板、参考图放到指定目录或直接附在任务里
4. Codex 根据本表读取 Obsidian 知识库，生成作图 Brief 和 GPT 提示词
5. 美工确认 Brief 后再进入 GPT 出图

> [!tip]
> 输入越具体，GPT 出图越稳定。尤其是产品材质、颜色、尺寸、禁止出现元素、参考模板，这几项会直接影响成图质量。

## Codex 处理规则

收到任务输入表后，Codex 应执行以下步骤：

1. 检查必填项是否完整
2. 读取相关规范：[[wiki/standards/亚马逊主图规范]]、[[wiki/standards/A+页面设计规范]]、[[wiki/standards/设计交付规范]]
3. 读取相关方法：[[wiki/concepts/AI生成电商主图]]、[[wiki/concepts/AI内容侵权风险与合规]]
4. 查找匹配模板、案例、产品资料
5. 生成标准作图 Brief
6. 生成 GPT 作图提示词
7. 输出 AI 自检清单

## 必填项检查

| 字段 | 为什么必须填 |
|------|--------------|
| 产品名称 | 用于识别任务对象 |
| ASIN / SKU | 用于归档、命名和追溯 |
| 图片类型 | 决定平台规范和提示词结构 |
| 目标站点 | 不同站点可能有语言和视觉差异 |
| 产品图片 | GPT 需要真实产品依据 |
| 核心卖点 | 决定画面表达重点 |
| 输出尺寸 | 决定构图和交付标准 |
| 禁止出现元素 | 降低合规和返工风险 |

## 常见缺失信息的追问

如果资料不完整，Codex 优先追问这些问题：

1. 这次图片用于哪里：主图、A+、广告，还是店铺页？
2. 有没有产品实拍图或精修图？
3. 产品最重要的 3 个卖点是什么？
4. 有没有必须套用的公司模板？
5. 有没有不能出现的元素、词语或竞品风格？
6. 是否需要英文文案？如果需要，文案由谁确认？

## 输出示例

Codex 最终应输出三份内容：

```markdown
## 1. 作图 Brief
整理后的设计需求，供美工确认。

## 2. GPT 作图提示词
可直接复制到 GPT 画图工具中执行。

## 3. AI 自检清单
用于检查产品一致性、平台规范、合规风险和交付要求。
```
